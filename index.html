Semana 04

MANEJO DEL DOM CON JAVASCRIPT Y CANVAS


¿Qué temas apredí?


Durante mi aprendizaje acerca de los tipos de variables en JavaScript, me familiaricé con var, let y const. La palabra clave var se utiliza para declarar variables globales o locales dentro de una función. Por otro lado, let se emplea para declarar variables cuyo valor puede cambiar a lo largo del programa, mientras que const se utiliza para variables cuyo valor permanece constante y no puede ser modificado una vez asignado. Además, exploré el uso de arrays para almacenar listas de elementos y objetos para estructurar datos más complejos. Los arrays son estructuras de datos que permiten almacenar múltiples valores bajo un mismo nombre, mientras que los objetos permiten agrupar propiedades y métodos relacionados.

También adquirí conocimientos sobre la ejecución de JavaScript en el navegador y cómo manipular el DOM (Document Object Model). Pude escribir y probar código JavaScript directamente en la consola del navegador, lo que me brindó resultados inmediatos y me permitió depurar y probar mi código de manera eficiente. Comprendí que el DOM es una representación en memoria de los elementos de una página web y permite interactuar con ellos. A través del DOM, pude modificar dinámicamente el contenido y la estructura de una página web, lo cual es esencial para crear experiencias interactivas y dinámicas en las aplicaciones web.

Estos conocimientos fundamentales en JavaScript me han brindado una base sólida para desarrollar aplicaciones web interactivas y dinámicas, permitiéndome manipular variables, utilizar estructuras de datos y interactuar con el DOM para crear experiencias más enriquecedoras para los usuarios.
Aprendí sobre el elemento canvas en HTML, una herramienta poderosa que permite crear gráficos, animaciones y visualizaciones interactivas directamente en un navegador web. Utilizando JavaScript, puedo dibujar formas, líneas y curvas en un área rectangular definida por el canvas. Este conocimiento me proporciona la capacidad de crear experiencias visuales dinámicas en sitios web, desde juegos interactivos hasta visualizaciones de datos complejas.
Reflexión:
Exploré JavaScript, comprendiendo variables (var, let, const) y estructuras de datos. Aprendí el manejo del DOM para interactividad web y funciones de flechas para código legible. Además, dominé el elemento canvas, permitiéndome crear gráficos y animaciones interactivas, mejorando mis habilidades en diseño web dinámico y visual.
Semana 05

DESARROLLO APLICACIONES DE FRONTEND CON FRAMEWORKS EN JS


¿Qué apredí?


Aprendí acerca de las Single Page Applications (SPAs) son una revolución en el mundo del desarrollo web. A diferencia de las aplicaciones tradicionales, las SPAs no requieren que los usuarios carguen una nueva página del servidor cada vez que interactúan con la aplicación. En cambio, una SPA carga una única página HTML en el navegador y luego utiliza JavaScript para manipular y actualizar el contenido en respuesta a las acciones del usuario.

Tambien aprendí acearce de React, una poderosa biblioteca de JavaScript desarrollada por Facebook, es una opción popular para crear SPAs. La magia de React radica en su enfoque de desarrollo basado en componentes. Cada parte de la interfaz de usuario se divide en componentes reutilizables y dinámicos, lo que facilita el mantenimiento y la expansión del código.


Tambien aprendí acerca de SASS (Syntactically Awesome Stylesheets) que es un preprocesador de CSS que permite escribir estilos de una manera más organizada y eficiente. Te permite utilizar características que no están disponibles en CSS puro, como variables, anidamiento, mixins y operaciones matemáticas, lo que simplifica el proceso de escritura y mantenimiento del código CSS.
React, una biblioteca de JavaScript, revoluciona el desarrollo web al permitir la creación de interfaces interactivas en aplicaciones de una sola página. Por otro lado, Next.js, un marco de desarrollo para React, lleva esta innovación un paso más allá. Al simplificar la creación de aplicaciones web tanto del lado del servidor como del cliente, Next.js se convierte en una herramienta esencial. Ofrece características como el enrutamiento del lado del servidor y la representación en el lado del servidor, facilitando así el desarrollo de aplicaciones React robustas y eficientes que brindan una experiencia de usuario excepcional.
Reflexión:
Aprendí sobre las Single Page Applications (SPAs), revolucionando la interactividad web sin cargar páginas del servidor constantemente. Exploré React, con su enfoque modular y dinámico, y SASS para estructurar estilos eficientemente. Además, Next.js amplió mis habilidades, permitiendo aplicaciones React versátiles y excepcionales en el servidor y el cliente.
Semana 06

EVENTOS, CONDICIONALES Y BUCLES CON FRAMEWORK JS


¿Qué apredí?


Aprendí acerca de Vite es una herramienta de construcción de aplicaciones web extremadamente rápida y flexible que ha ganado popularidad en el ecosistema de React. A diferencia de otras herramientas de construcción como Webpack, Vite se destaca por su velocidad y eficiencia, especialmente durante el proceso de desarrollo.

Vite se integra sin problemas con React, permitiendo a los desarrolladores aprovechar sus características avanzadas mientras trabajan con la biblioteca de React. Al usar Vite con React, los desarrolladores pueden disfrutar de un entorno de desarrollo instantáneo que proporciona recargas en caliente (hot module replacement), lo que significa que los cambios en el código se reflejan inmediatamente en el navegador sin necesidad de recargar toda la aplicación. Esto mejora significativamente la eficiencia y la velocidad del flujo de trabajo de desarrollo.

Además, Vite es configurado por defecto para aprovechar las últimas características de ECMAScript (ES modules) y proporciona una estructura de proyectos clara y sencilla. Los desarrolladores pueden disfrutar de un desarrollo rápido y eficiente, lo que lo convierte en una opción atractiva para proyectos de React. En resumen, Vite es una herramienta de construcción extremadamente rápida y eficiente que se integra sin problemas con React, proporcionando un entorno de desarrollo ágil y productivo para los desarrolladores de aplicaciones web basadas en React.
Reflexión:
Vite es una herramienta veloz y flexible para construir aplicaciones web React, destacando por sus recargas instantáneas y perfecta integración. Su configuración predeterminada para ECMAScript y estructura clara lo hacen ideal, mejorando la eficiencia del desarrollo. En síntesis, Vite ofrece un entorno de desarrollo ágil y productivo para proyectos React.

Semana 09

ROUTER MEMO CSS REACT


¿Qué apredí?


React Router me ha permitido gestionar la navegación en mis aplicaciones React de manera eficiente. Puedo crear rutas y definir qué componente se debe renderizar según la URL actual. Esto es especialmente útil en aplicaciones de una sola página (SPA) donde la navegación se realiza sin tener que recargar toda la página. Con React Router, puedo establecer parámetros en las URL, navegar entre diferentes páginas y acceder a esos parámetros en los componentes. Esto ha ampliado las posibilidades de interacción en mis aplicaciones y ha brindado una experiencia de usuario más fluida.

React.memo ha sido una herramienta muy útil para optimizar el rendimiento de mis componentes funcionales en React. Al envolver un componente con React.memo, puedo evitar su re-renderización innecesaria. React.memo compara las propiedades del componente y solo lo renderiza nuevamente si las propiedades han cambiado. Esto ha mejorado significativamente el rendimiento de mis aplicaciones al reducir la cantidad de renderizaciones innecesarias. Es especialmente útil para componentes que reciben propiedades estáticas o inmutables, ya que evita gastos de rendimiento innecesarios y mejora la fluidez de la aplicación.


En cuanto a CSS en React, he aprendido diferentes enfoques para aplicar estilos a mis componentes. Puedo utilizar CSS tradicional importando archivos CSS externos y aplicándolos a mis componentes. También tengo la opción de usar CSS en línea dentro de los componentes utilizando la propiedad de estilo en JSX. Además, he descubierto bibliotecas populares como Styled Components o CSS Modules, que permiten escribir CSS de forma más modular y encapsulada dentro de los componentes. Estas bibliotecas me han brindado flexibilidad y mejor organización a la hora de trabajar con estilos en mis proyectos de React.
Reflexión:
gracias a React Router, React.memo y CSS en React, he mejorado mi capacidad para crear aplicaciones web interactivas y eficientes. Ahora puedo gestionar la navegación de manera efectiva, optimizar el rendimiento de mis componentes y aplicar estilos de forma más modular y estructurada. Estas herramientas me han permitido desarrollar aplicaciones más sólidas, mejorar la experiencia del usuario y optimizar el rendimiento general de mis proyectos en React.

INTRODUCCION A NEXTJS


¿Qué aprendí?


Al aprender sobre Next.js, he adquirido conocimientos valiosos para desarrollar aplicaciones web modernas y escalables. Next.js es un framework de React que proporciona características adicionales y facilita la creación de aplicaciones de una sola página (SPA) y aplicaciones de renderizado del lado del servidor (SSR).



Una de las principales cosas que he aprendido sobre Next.js es su capacidad para el enrutamiento dinámico y estático. Puedo definir rutas personalizadas y manejar parámetros en la URL de una manera sencilla y flexible. Además, Next.js me permite generar páginas estáticas previamente renderizadas en tiempo de compilación, lo que mejora el rendimiento y la velocidad de carga de mi aplicación.



Otra característica destacada de Next.js es su capacidad para el renderizado del lado del servidor (SSR). Esto significa que puedo renderizar mis páginas en el servidor antes de enviarlas al cliente, lo que permite una carga inicial más rápida y una mejor optimización para los motores de búsqueda. Además, puedo aprovechar la función de generación estática y dinámica de Next.js para equilibrar la carga en el servidor y mejorar la experiencia del usuario.

Next.js también proporciona una fácil integración con API y servicios externos. Puedo crear rutas API personalizadas y manejar solicitudes y respuestas de manera eficiente. Esto me permite desarrollar aplicaciones web interactivas que se comunican con servidores y bases de datos externos.

Reflexión:
Al aprender sobre Next.js, he adquirido habilidades fundamentales para desarrollar aplicaciones web modernas y escalables. He aprovechado su enrutamiento dinámico y estático, el renderizado del lado del servidor, la integración de API y servicios externos, así como sus herramientas y bibliotecas adicionales. Next.js ha mejorado mi capacidad para crear aplicaciones web de alto rendimiento y brindar una experiencia excepcional a los usuarios.

Semana 13

FRAMEWORK SPRING


¿Qué apredí?


Al aprender sobre el framework Spring, he adquirido conocimientos valiosos que han mejorado mi capacidad para desarrollar aplicaciones web y empresariales de manera eficiente y escalable. Spring es un framework de desarrollo de aplicaciones Java que proporciona una amplia gama de características y beneficios.

Una de las principales cosas que he aprendido sobre Spring es su enfoque en la inversión de control (IoC) y la inyección de dependencias (DI). Esto significa que puedo diseñar mis aplicaciones de manera modular, separando las diferentes capas y componentes, y permitiendo que Spring se encargue de la gestión de las dependencias. La inyección de dependencias me ha permitido escribir código más limpio, fácil de mantener y altamente testeable.


Además, Spring proporciona un conjunto completo de herramientas y módulos para el desarrollo de aplicaciones empresariales. Por ejemplo, Spring MVC ofrece un modelo de desarrollo basado en controladores que facilita la creación de API RESTful y aplicaciones web. Spring Data proporciona una capa de abstracción para interactuar con diferentes sistemas de almacenamiento de datos, como bases de datos relacionales o NoSQL. Spring Security ofrece una solución completa de autenticación y autorización para proteger mis aplicaciones.
Otro aspecto importante que he aprendido sobre Spring es su integración con otros frameworks y tecnologías populares en el ecosistema Java. Spring se integra bien con Hibernate para el mapeo objeto-relacional, con Apache Kafka para la mensajería y con otras herramientas como Redis, Elasticsearch y RabbitMQ. Esto me ha permitido aprovechar al máximo el ecosistema de Java y utilizar las mejores herramientas para desarrollar aplicaciones robustas y escalables.
Reflexión:
